# 教训与约定

- **大规模 Java 代码库**：「每个函数一篇说明」应通过 IDE、Javadoc、或自动索引解决；维护手写清单会与上游快速漂移。
- **生成文档的路径**：索引中应使用相对仓库根的路径或环境变量说明，避免把某台机器的 `/tmp/...` 写死进版本库。
- **超大自动生成 Markdown**：按源码一级目录分卷 + 单独 README 索引，比单文件上万行更易打开、diff 与检索。
- **精读文档插入新章节**：在中间插入编号小节会顺延后续节号；若外部文档引用「第 N 节」，合并后需同步检索替换。
- **精读文末路线图表格**：插入节后同步更新「后续路线」表里的编号与「下一篇」条目，避免读者仍指向旧节号。
- **经济链路用词**：`PlayerShop` 与雇佣商店均属迷你房间形态；正文用语统一用「摆摊」等与源码注释一致的写法，避免与真实地名谐音混淆。
