## 服务端部署

### 1. 构建生产版本

```powershell
# 进入项目目录
cd BeiDou-Server/gms-server

# Maven 构建（跳过测试）
mvn clean package -DskipTests

# 输出文件
target/BeiDou.jar
```

### 2. 生产环境配置

创建生产配置文件 `application-prod.yml`：

```yaml
server:
  port: 8686

jwt:
  # 生产环境必须修改为自定义UUID
  secret: "your-custom-uuid-here"
  duration: 1800000

mybatis-flex:
  datasource:
    mysql:
      url: jdbc:mysql://your-db-host:3306/beidou?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
      username: beidou_user
      password: your-db-password

springdoc:
  api-docs:
    enabled: false   # 生产环境关闭
  swagger-ui:
    enabled: false   # 生产环境关闭

gms:
  service:
    language: zh-CN
    rate-limit:
      enabled: true   # 生产环境开启限流
      limit: 100
      duration: 60000
      auto-ban: false
    wan-host: your-public-ip
    lan-host: your-lan-ip
    localhost: 127.0.0.1
    login-port: 8484
```

### 3. 启动服务端

**Linux 启动脚本：**

```bash
#!/bin/bash
# start.sh

JAVA_HOME=/path/to/jdk-21
APP_NAME=BeiDou.jar
LOG_FILE=logs/beidou.log

nohup $JAVA_HOME/bin/java -jar $APP_NAME --spring.profiles.active=prod > $LOG_FILE 2>&1 &
echo $! > beidou.pid
echo "BeiDou Server started, PID: $(cat beidou.pid)"
```

**Windows 启动命令：**

```powershell
java -jar BeiDou.jar --spring.profiles.active=prod
```

### 4. JVM优化配置

```bash
java -Xms4g -Xmx8g -XX:+UseG1GC -XX:MaxGCPauseMillis=200 -jar BeiDou.jar
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| -Xms4g | 初始堆内存4GB |
| -Xmx8g | 最大堆内存8GB |
| -XX:+UseG1GC | 使用G1垃圾回收器 |
| -XX:MaxGCPauseMillis=200 | GC最大暂停时间200ms |

## 前端部署

### 1. 构建生产版本

```powershell
cd BeiDou-Server/gms-ui

# 安装依赖
yarn install

# 构建
yarn build

# 输出目录
dist/
```

### 2. 部署方式

**方式一：Nginx部署**

```nginx
# nginx.conf
server {
    listen 8787;
    server_name your-domain.com;
    
    root /path/to/gms-ui/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API反向代理
    location /api {
        proxy_pass http://localhost:8686;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**方式二：静态文件服务**

将 `dist/` 目录放到任意Web服务器。

### 3. API地址配置

修改前端配置，指向生产环境API：

```typescript
// src/config/settings.json
{
  "apiBaseUrl": "http://your-api-server:8686"
}
```

## 数据库部署

### 1. MySQL生产配置

```ini
# my.cnf / my.ini

[mysqld]
# 基本配置
port = 3306
bind-address = 127.0.0.1

# 性能配置
max_connections = 500
innodb_buffer_pool_size = 4G
innodb_log_file_size = 512M
innodb_flush_log_at_trx_commit = 2

# 字符集
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci

# 安全配置
skip-name-resolve
local_infile = OFF
```

### 2. 创建生产用户

```sql
-- 创建数据库用户
CREATE USER 'beidou_user'@'localhost' IDENTIFIED BY 'your-password';

-- 授权
GRANT ALL PRIVILEGES ON beidou.* TO 'beidou_user'@'localhost';

-- 授权额外权限（如需要）
GRANT SELECT ON performance_schema.user_variables_by_thread TO 'beidou_user'@'localhost';
GRANT SHOW VIEW ON mysql.* TO 'beidou_user'@'localhost';

FLUSH PRIVILEGES;
```

### 3. 数据库安全

- 使用强密码
- 限制远程访问
- 定期更新密码
- 监控异常查询

## 安全配置

### 1. JWT密钥安全

**生产环境必须：**

```yaml
jwt:
  secret: "your-custom-uuid-here"  # 使用自定义UUID
```

生成UUID：

```powershell
# PowerShell
[guid]::NewGuid().ToString()

# Java
UUID.randomUUID().toString()
```

### 2. Swagger关闭

生产环境关闭 Swagger 防止接口泄露：

```yaml
springdoc:
  api-docs:
    enabled: false
  swagger-ui:
    enabled: false
```

### 3. 限流配置

生产环境建议开启API限流：

```yaml
gms:
  service:
    rate-limit:
      enabled: true
      limit: 100        # 每IP每分钟100次请求
      duration: 60000
      auto-ban: false   # 根据需要决定是否自动封禁
```

### 4. HTTPS配置

建议使用HTTPS加密通信：

**方式一：Nginx SSL**

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # ... 其他配置
}
```

**方式二：Spring Boot SSL**

```yaml
server:
  port: 8686
  ssl:
    enabled: true
    key-store: /path/to/keystore.p12
    key-store-password: your-password
    key-store-type: PKCS12
```

### 5. 防火墙配置

开放必要端口：

| 端口 | 服务 | 是否对外开放 |
|------|------|--------------|
| 8484 | 游戏客户端 | 是 |
| 8686 | REST API | 根据需要 |
| 8787 | Web前端 | 是 |
| 3306 | MySQL | 否（仅内网） |

**Linux防火墙示例：**

```bash
# 开放游戏端口
firewall-cmd --add-port=8484/tcp --permanent
firewall-cmd --add-port=8787/tcp --permanent

# MySQL仅允许内网访问
firewall-cmd --add-rich-rule='rule family="ipv4" source address="127.0.0.1" port port="3306" protocol="tcp" accept' --permanent

# 重载防火墙
firewall-cmd --reload
```

## Docker部署

BeiDou 已将 Docker 配置独立到专门仓库：

https://github.com/BeiDouMS/BeiDou-docker

### Docker镜像

可直接拉取镜像：

```bash
docker pull ghcr.io/beidoums/beidou-server-all:latest
```

### Docker Compose部署

```yaml
# docker-compose.yml
version: '3'
services:
  mysql:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: beidou
    volumes:
      - mysql-data:/var/lib/mysql
    ports:
      - "3306:3306"
    
  server:
    image: ghcr.io/beidoums/beidou-server-all:latest
    depends_on:
      - mysql
    ports:
      - "8484:8484"
      - "8686:8686"
    
  ui:
    image: nginx:alpine
    volumes:
      - ./gms-ui/dist:/usr/share/nginx/html
    ports:
      - "8787:80"

volumes:
  mysql-data:
```

### 启动容器

```bash
docker-compose up -d
```

## 监控与日志

### 1. 日志配置

服务端使用 Log4j2，日志配置位于 `log4j2.xml`。

**日志级别：**

| 环境 | 推荐级别 |
|------|----------|
| 开发 | DEBUG |
| 生产 | INFO |

### 2. 日志位置

```bash
logs/
├── beidou.log        # 主日志
├── beidou-error.log  # 错误日志
└── beidou-access.log # 访问日志
```

### 3. 日志轮转

建议配置日志轮转防止日志文件过大：

```xml
<RollingFile name="RollingFile" fileName="logs/beidou.log"
             filePattern="logs/beidou-%d{yyyy-MM-dd}-%i.log">
    <PatternLayout pattern="%d{HH:mm:ss.SSS} [%t] %-5level %logger{36} - %msg%n"/>
    <Policies>
        <TimeBasedTriggeringPolicy interval="1" modulate="true"/>
        <SizeBasedTriggeringPolicy size="100MB"/>
    </Policies>
    <DefaultRolloverStrategy max="10"/>
</RollingFile>
```

### 4. 监控指标

关键监控指标：

| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| CPU使用率 | 服务器CPU负载 | >80% |
| 内存使用率 | JVM内存使用 | >85% |
| 连接数 | 在线玩家数 | 根据容量 |
| 响应时间 | API响应时间 | >500ms |
| 错误率 | API错误比例 | >1% |

## 备份策略

### 1. 数据库备份

**每日自动备份脚本：**

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR=/backup/mysql
DATE=$(date +%Y%m%d)
DB_NAME=beidou
DB_USER=root
DB_PASS=your-password

# 备份
mysqldump -u$DB_USER -p$DB_PASS $DB_NAME > $BACKUP_DIR/$DB_NAME_$DATE.sql

# 压缩
gzip $BACKUP_DIR/$DB_NAME_$DATE.sql

# 保留最近30天
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete

echo "Backup completed: $DB_NAME_$DATE.sql.gz"
```

### 2. 配置备份

定期备份以下配置：
- application.yml
- config.ini
- game_config 表数据

### 3. 备份验证

定期验证备份可用性：

```bash
# 恢复测试（测试环境）
mysql -u root -p test_beidou < backup_file.sql
```

## 故障排查

### 1. 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 服务无法启动 | 端口占用 | 检查端口，修改配置 |
| 数据库连接失败 | 配置错误 | 检查用户名密码 |
| 客户端无法连接 | IP配置错误 | 检查wan-host配置 |
| API返回401 | Token失效 | 重新登录获取Token |
| 游戏卡顿 | 内存不足 | 增加JVM内存 |

### 2. 排查步骤

**服务端排查：**

```bash
# 检查进程
ps aux | grep BeiDou

# 检查端口
netstat -tlnp | grep 8484
netstat -tlnp | grep 8686

# 检查日志
tail -f logs/beidou-error.log
```

**数据库排查：**

```bash
# 检查MySQL状态
mysqladmin -u root -p status

# 检查连接数
mysql -e "SHOW PROCESSLIST;"

# 检查表状态
mysql -e "SHOW TABLE STATUS FROM beidou;"
```

### 3. 性能优化

**JVM调优：**

```bash
# 查看JVM状态
jstat -gc <pid>

# 生成堆分析
jmap -histo <pid> > heap.txt

# 导出堆转储
jmap -dump:format=b,file=heap.bin <pid>
```

**数据库调优：**

```sql
-- 查看慢查询
SHOW VARIABLES LIKE 'slow_query%';

-- 查看表索引
SHOW INDEX FROM characters;

-- 分析表
ANALYZE TABLE characters;
```

## 运维流程

### 启动流程

1. 启动MySQL服务
2. 启动服务端（gms-server）
3. 启动前端（gms-ui / Nginx）
4. 验证服务状态

### 停止流程

1. 通知在线玩家
2. 停止前端服务
3. 停止服务端
4. 执行数据库备份

### 更新流程

1. 备份数据库和配置
2. 构建新版本
3. 停止旧版本服务
4. 部署新版本
5. 验证服务状态
6. 监控运行情况
