## 服务端环境 (gms-server)

### 系统要求

- 操作系统：Windows 10/11 或 Linux
- Java 版本：OpenJDK 21
- 内存：建议 8GB 以上
- 磁盘：建议 20GB 以上

### 1. 安装 OpenJDK 21

**Windows 安装步骤：**

1. 下载 OpenJDK 21：https://jdk.java.net/archive/
2. 解压到目标目录，例如 `C:\Program Files\Java\jdk-21`
3. 配置环境变量：
   - `JAVA_HOME`: `C:\Program Files\Java\jdk-21`
   - `Path`: 添加 `%JAVA_HOME%\bin`

**验证安装：**

```powershell
java -version
# 输出应显示 openjdk version "21.x.x"
```

### 2. 安装 Maven

1. 下载 Maven：https://maven.apache.org/download.cgi
2. 解压到目标目录
3. 配置环境变量：
   - `MAVEN_HOME`: Maven 解压路径
   - `Path`: 添加 `%MAVEN_HOME%\bin`

**验证安装：**

```powershell
mvn -version
```

### 3. IntelliJ IDEA 配置

1. 安装 IntelliJ IDEA 2023.3 或更高版本：https://www.jetbrains.com/idea/
2. 打开 IDEA，选择 `File > Open`
3. 选择 `BeiDou-Server/gms-server` 目录或根 pom.xml 文件
4. 等待 Maven 自动下载依赖

**重要配置：**

如果直接打开 Cosmic 目录运行，需要在 Server 的编译配置里设置 Working Directory 为 `gms-server`。

### 4. 项目构建

```powershell
# 进入项目目录
cd BeiDou-Server/gms-server

# Maven 构建
mvn clean package -DskipTests

# 输出文件位置
# target/BeiDou.jar
```

### 5. 运行服务端

```powershell
# 直接运行 jar 文件
java -jar target/BeiDou.jar

# 或在 IDEA 中运行 ServerApplication 主类
```

## 前端环境 (gms-ui)

### 系统要求

- Node.js 版本：v20.15.0 LTS
- 操作系统：Windows 10/11、macOS 或 Linux

### 1. 安装 Node.js

**Windows 安装步骤：**

1. 下载 Node.js v20.15.0：https://nodejs.org/dist/v20.15.0/node-v20.15.0-x64.msi
2. 运行安装程序，按照提示完成安装
3. 验证安装：

```powershell
node -v
# 输出: v20.15.0

npm -v
# 输出 npm 版本号
```

### 2. 安装 Yarn

```powershell
npm install -g yarn
```

**验证安装：**

```powershell
yarn -v
```

> 如果提示 npm 命令不存在，可能是安装 Node.js 时环境变量未生效，重启系统即可。

### 3. 初始化项目

```powershell
# 进入前端目录
cd BeiDou-Server/gms-ui

# 安装依赖
yarn install
```

### 4. 启动开发服务器

```powershell
# 开发模式
yarn dev

# 默认访问地址
# http://localhost:8787
```

### 5. 构建生产版本

```powershell
# 构建
yarn build

# 输出目录
# dist/
```

### 常用命令

| 命令 | 描述 |
|------|------|
| `yarn dev` | 启动开发服务器 |
| `yarn build` | 构建生产版本 |
| `yarn preview` | 预览构建结果 |
| `yarn type:check` | TypeScript 类型检查 |

## 客户端DLL环境 (ijl15)

### 系统要求

- Visual Studio 2019
- Windows SDK 10
- 工具集：VS2019 (v142)
- 平台：x86 (32位)

### 1. 安装 Visual Studio 2019

1. 下载 Visual Studio 2019：https://visualstudio.microsoft.com/
2. 安装时选择以下组件：
   - "使用 C++ 的桌面开发" 工作负载
   - Windows 10 SDK
   - MSVC v142 - VS 2019 C++ x86/x64 构建工具

### 2. 打开项目

1. 使用 VS 2019 打开 `BeiDou-ijl15/ezorsia/ezorsia.vcxproj`
2. 或打开解决方案文件（如存在）

### 3. 配置构建模式

**必须使用 Release x86 模式：**

1. 在 Visual Studio 顶部工具栏找到配置下拉菜单
2. 选择 `Release`
3. 在平台下拉菜单选择 `x86` (或 `Win32`)

### 4. 生成解决方案

1. 菜单栏选择 `Build > Build Solution`
2. 或使用快捷键 `Ctrl+Shift+B`

### 5. 输出文件位置

构建完成后，输出文件位于：

```
BeiDou-ijl15/out/Release/ijl15.dll
```

## 数据库环境

### 系统要求

- MySQL 8.x（不支持 MySQL 8 以下版本）

### 1. 安装 MySQL 8

**下载地址：**

- 官方下载：https://downloads.mysql.com/archives/community/
- 工具推荐：https://github.com/SleepNap/NapMysqlTool/releases/latest

**安装步骤：**

1. 运行安装程序
2. 设置 root 密码（开发环境可设置为 `root`）
3. 配置 MySQL 服务自动启动

### 2. 数据库管理工具

推荐使用以下工具：

- DBeaver：https://dbeaver.io/download/
- Navicat Lite：https://www.navicat.com/en/download/navicat-premium-lite

### 3. 数据库初始化

服务端使用 Flyway 自动管理数据库迁移，启动服务端时会：

1. 自动创建 `beidou` 数据库
2. 执行所有初始化 SQL 脚本
3. 创建必要的表结构和初始数据

只需确保 MySQL 服务已启动即可。

### 4. 数据库用户权限

如果数据库用户不是 root 用户，需要以下额外权限：

- `performance_schema` 库 `user_variables_by_thread` 表的 SELECT 权限
- `mysql` 库的 SHOW VIEW 权限

## 常见问题

### Q: Maven 下载依赖很慢怎么办？

A: 项目已配置阿里云镜像仓库，位于 pom.xml 中：

```xml
<repository>
    <id>central</id>
    <url>https://maven.aliyun.com/repository/public</url>
</repository>
```

### Q: IDEA 提示找不到主类？

A: 确保 Working Directory 设置正确。如果直接打开 Cosmic 目录，需要设置 Working Directory 为 `gms-server`。

### Q: 前端 yarn install 报错？

A: 检查 Node.js 版本是否为 v20.15.0，尝试删除 `node_modules` 目录后重新执行 `yarn install`。

### Q: Visual Studio 构建失败？

A: 确保：
1. 使用 Release x86 模式（不是 x64）
2. 工具集设置为 VS2019 (v142)
3. Windows SDK 10 已正确安装

### Q: 数据库连接失败？

A: 检查 `application.yml` 配置：

```yaml
mybatis-flex:
  datasource:
    mysql:
      url: jdbc:mysql://localhost:3306/beidou?...
      username: root
      password: root
```

确保 MySQL 服务已启动，用户名密码正确。

### Q: 服务端启动后数据库表没有创建？

A: Flyway 会自动创建表，检查：
1. MySQL 是否已启动
2. 数据库连接配置是否正确
3. Flyway 配置是否启用（默认启用）
