# 阅读指引：文档分层与推荐路径

本站文档种类较多，先弄清**分层**再搜关键字，会省时很多。

!!! tip "优先想「理解系统在干什么」"

    请先看 **[源码模块精读（人机撰写）](12-源码模块精读-人机撰写.md)**。这里的段落是顺着调用链、数据结构写的说明，**不是**脚本根据命名猜出来的空话。

---

## 三类文档（怎么选）

| 类型 | 典型路径 | 适合你吗 |
|------|-----------|-----------|
| **叙述 / 精读** | `09` 架构导读、`10` ijl15 详解、`11` 生成说明、`12` 模块精读 | 想搞懂 **流程、边界、为何这么设计** |
| **符号清单（全量）** | `reference/*-full-symbols.md`、`gms-server-api-index.md` | 已知类名/方法名，要 **快速跳到源码位置** |
| **逐符号一句话（启发式）** | `reference/gms-server-symbol-notes*`、`gms-ui-symbol-notes` 等 | **仅辅助检索**；机械摘要易泛化，勿当设计文档 |

!!! warning "自动生成的「职责」行"

    `*-symbol-notes` 由脚本按命名规则批量生成。若出现大量空段、计数为 0 或与实现不符，请 **直接以源码为准**，不必纠结该行解释。

---

## 推荐阅读顺序（后端 / 客户端大致各一条）

1. **[源码深度导读](09-源码深度导读.md)**：包结构、Netty 与 Spring 分工、从哪读起。  
2. **[源码模块精读](12-源码模块精读-人机撰写.md)**：按章节跟进（REST、登录、`PlayerLoggedin`、地图与 Portal 等）。  
3. 需要查具体类时：打开 **[gms-server public 索引](reference/gms-server-api-index.md)** 或 **全量符号**，配合 IDE 跳转。  
4. 客户端 DLL：**[ijl15 源码详解](10-BeiDou-ijl15-源码详解.md)**（体量小，可按文件读完）。

---

## 站内技巧

- 顶部 **搜索**：跨页面全文检索（含自动生成的大文件）。  
- **目录**：长文右侧（或集成在导航中）可 **跟随滚动**，便于在大章节间跳转。  
- **深色 / 浅色**：页眉调色盘图标切换。

若你发现某段精读与源码不一致，欢迎在开 issue 时附上 **文件路径 + 行号或提交**，便于对照修订。
