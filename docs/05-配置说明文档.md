## 服务端配置

### 配置文件位置

```
gms-server/src/main/resources/application.yml
```

### 完整配置说明

```yaml
server:
  port: 8686                      # API服务端口

app:
  vue: http://localhost:8787      # 前端地址，用于跨域配置
  
jwt:
  secret: "50da066e-6080-40f5-a173-86bd27d4f674"  # JWT签名密钥
  duration: 1800000               # Token有效期（毫秒），默认30分钟

mybatis-flex:
  datasource:
    mysql:
      type: com.alibaba.druid.pool.DruidDataSource  # 数据源类型
      driver-class-name: com.mysql.cj.jdbc.Driver   # 驱动类
      url: jdbc:mysql://localhost:3306/beidou?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai  # 数据库连接URL
      username: root               # 数据库用户名
      password: root               # 数据库密码
  global-config:
    print-banner: false           # 是否打印MyBatis-Flex banner

springdoc:
  api-docs:
    enabled: true                 # 是否开启OpenApi
  swagger-ui:
    enabled: true                 # 是否开启SwaggerUI

spring:
  flyway:
    validate-on-migrate: false    # 禁用迁移版本验证
  servlet:
    multipart:
      max-file-size: 1MB          # 单文件大小限制
      max-request-size: 10MB      # 总请求大小限制
  jackson:
    time-zone: Asia/Shanghai      # JSON时区
    date-format: yyyy-MM-dd HH:mm:ss  # JSON日期格式

gms:
  service:
    language: zh-CN               # 服务语言（zh-CN、en-US）
    rate-limit:
      enabled: false              # 是否开启限流
      limit: 10                    # 每IP最大请求数
      duration: 1000              # 重置时间（毫秒）
      auto-ban: false              # 是否自动封禁
    wan-host: 127.0.0.1           # 公网IP
    lan-host: 127.0.0.1           # 局域网IP
    localhost: 127.0.0.1          # 本地IP
    login-port: 8484              # 游戏登录端口
```

### 配置分类详解

#### 1. 服务器端口配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| server.port | 8686 | REST API 服务端口 |
| gms.service.login-port | 8484 | 游戏客户端连接端口 |

#### 2. JWT认证配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| jwt.secret | UUID字符串 | Token签名密钥 |
| jwt.duration | 1800000 | Token有效期（30分钟） |

**安全提醒：** 生产环境必须修改 jwt.secret 为自定义 UUID。

#### 3. 数据库配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| url | jdbc:mysql://localhost:3306/beidou | 数据库连接地址 |
| username | root | 数据库用户名 |
| password | root | 数据库密码 |

**连接参数说明：**

| 参数 | 说明 |
|------|------|
| useUnicode=true | 使用Unicode编码 |
| characterEncoding=utf-8 | 字符编码UTF-8 |
| useSSL=false | 不使用SSL连接 |
| serverTimezone=Asia/Shanghai | 时区设置 |

#### 4. Swagger配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| springdoc.api-docs.enabled | true | 开启OpenApi |
| springdoc.swagger-ui.enabled | true | 开启SwaggerUI |

**生产环境建议关闭：**

```yaml
springdoc:
  api-docs:
    enabled: false
  swagger-ui:
    enabled: false
```

#### 5. 多语言配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| gms.service.language | zh-CN | 服务语言 |

**可选值：**
- `zh-CN` - 中文简体
- `en-US` - 英文

#### 6. 限流配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| rate-limit.enabled | false | 是否开启限流 |
| rate-limit.limit | 10 | 每IP最大请求数 |
| rate-limit.duration | 1000 | 重置时间（毫秒） |
| rate-limit.auto-ban | false | 是否自动封禁 |

#### 7. IP配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| wan-host | 127.0.0.1 | 公网IP |
| lan-host | 127.0.0.1 | 局域网IP |
| localhost | 127.0.0.1 | 本地IP |

## 客户端DLL配置

### 配置文件位置

```
BeiDou-ijl15/ezorsia/config.ini
```

将生成的 ijl15.dll 和 config.ini 复制到游戏客户端目录。

### 完整配置说明

```ini
[general]
;IME 修复方案，只有0和1两种，优先尝试方案1（更完美），如果会卡门再尝试方案0
imeType=1
;分辨率 (推荐值: 1280x720, 1366x768; 其他可选: 1600x900, 1920x1080, 1024x768; 不保证可用: 800x600)
width=1280
height=720
;服务器地址
ServerIP_Address=127.0.0.1
serverIP_Port=8484
;加载文字汉化
SwitchChinese=true
;右下角消息数量
MsgAmount=10
;是否包含EzorsiaV2 WZ文件
EzorsiaV2WzIncluded=true
;是否使用自定义登录框
CustomLoginFrame=true
;开启后登录界面的UI会按大边框布局
bigLoginFrame=true
;窗口模式
WindowedMode=true
;跳过开头的动画
RemoveLogos=true
;是否启用VirtuProtect
UseVirtuProtect=true

[optional]
;物理攻击面板上限
setDamageCap=199999
;魔攻/魔防面板上限
setMAtkCap=1999
;命中上限
setAccCap=999
;回避上限
setAvdCap=999
;真实输出上限
setAtkOutCap=199999
;移速上限
speedMovementCap=140
;跳跃上限
jumpCap=123
;爬绳速度自适应
climbSpeedAuto=false
;爬绳基础速度系数
climbSpeed=1.0
;超级机制
useTubi=true
;是否使用自定义登录框
ownLoginFrame=false
;是否使用自定义商城背景
ownCashShopFrame=false
;允许重复发言
talkRepeat=false
;连续说话限制（毫秒）
talkTime=2000

[debug]
debug=false
;免密模式，须先开启debug
noPassword=false
```

### 配置分类详解

#### 1. 基本配置 [general]

| 参数 | 默认值 | 说明 |
|------|--------|------|
| iimeType | 1 | IME输入法修复方案（0或1） |
| width | 1280 | 分辨率宽度 |
| height | 720 | 分辨率高度 |
| ServerIP_Address | 127.0.0.1 | 服务器地址 |
| serverIP_Port | 8484 | 服务器端口 |
| SwitchChinese | true | 加载文字汉化 |
| MsgAmount | 10 | 右下角消息数量 |
| WindowedMode | true | 窗口模式 |
| RemoveLogos | true | 跳过开头动画 |
| UseVirtuProtect | true | 启用VirtuProtect |

**IME修复方案说明：**

| 值 | 说明 |
|----|------|
| 0 | 方案0，较基础 |
| 1 | 方案1，更完美，推荐优先尝试 |

如果方案1会导致"卡门"问题，请尝试方案0。

**推荐分辨率：**

| 分辨率 | 状态 |
|--------|------|
| 1280x720 | 推荐 |
| 1366x768 | 推荐 |
| 1600x900 | 可用 |
| 1920x1080 | 可用 |
| 1024x768 | 可用 |
| 800x600 | 不保证可用 |

#### 2. 属性上限配置 [optional]

| 参数 | 默认值 | 说明 |
|------|--------|------|
| setDamageCap | 199999 | 物理攻击面板上限 |
| setMAtkCap | 1999 | 魔攻/魔防面板上限 |
| setAccCap | 999 | 命中上限 |
| setAvdCap | 999 | 回避上限 |
| setAtkOutCap | 199999 | 真实输出上限 |
| speedMovementCap | 140 | 移速上限 |
| jumpCap | 123 | 跳跃上限 |

**功能说明：** 突破原版游戏的属性上限限制。

#### 3. 其他功能配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| climbSpeedAuto | false | 爬绳速度自适应 |
| climbSpeed | 1.0 | 爬绳基础速度系数 |
| useTubi | true | 超级机制（解决升级停顿、宠物攻击中断） |
| talkRepeat | false | 允许重复发言 |
| talkTime | 2000 | 连续说话限制（毫秒） |

#### 4. 调试配置 [debug]

| 参数 | 默认值 | 说明 |
|------|--------|------|
| debug | false | 开启调试模式 |
| noPassword | false | 免密模式（须先开启debug） |

**免密模式说明：**
- 须先开启 debug=true
- 需要服务端配合支持
- 客户端仅解除密码限制

## 前端配置

### 配置文件位置

```
gms-ui/src/config/settings.json
```

### 主要配置项

```json
{
  "title": "BeiDou 管理后台",
  "version": "1.0.0",
  "apiBaseUrl": "http://localhost:8686"
}
```

### Vite配置

配置文件位于 `config/vite.config.dev.ts` 和 `config/vite.config.prod.ts`。

### package.json配置

```json
{
  "scripts": {
    "dev": "vite --config ./config/vite.config.dev.ts",
    "build": "vue-tsc --noEmit && vite build --config ./config/vite.config.prod.ts"
  }
}
```

## 配置修改方式

### 服务端配置修改

1. 编辑 `application.yml` 文件
2. 重启服务端生效

### 客户端配置修改

1. 编辑游戏客户端目录下的 `config.ini`
2. 重新启动游戏客户端生效

### 游戏参数修改

方式一：数据库直接修改

```sql
UPDATE game_config 
SET config_value = '2.0' 
WHERE config_type = 'world' 
AND config_sub_type = '0' 
AND config_code = 'exp_rate';
```

方式二：Web后台修改
- 登录 gms-ui 管理后台
- 进入游戏配置页面
- 修改参数值
- 保存后自动生效
