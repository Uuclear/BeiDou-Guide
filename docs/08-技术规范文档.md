## API版本控制规范

### 版本机制

API 采用 URL 路径版本控制：

```
/v1/resource
/v2/resource
/v3/resource
```

### 版本常量

定义版本常量，便于统一管理：

```java
public class ApiConstant {
    public static final String V1 = "v1";
    public static final String V2 = "v2";
    public static final String V3 = "v3";
    public static final String LATEST = "v3";  // 当前最新版本
}
```

### 控制器标注

所有控制器必须标注版本：

```java
@RestController
@RequestMapping("/" + ApiConstant.LATEST + "/characters")
@Tag(name = ApiConstant.LATEST)
public class CharacterController {
    // ...
}
```

### 版本更新规则

| 场景 | 处理方式 | 示例 |
|------|----------|------|
| 新增接口 | 使用 LATEST | 新增接口直接使用 `ApiConstant.LATEST` |
| 兼容修改 | 更新 LATEST | 小改动直接更新现有版本 |
| 不兼容修改 | 新建版本 | 旧接口改为指定版本，新接口使用新版本 |
| 版本切换 | 更新 LATEST 常量 | 将 `LATEST` 指向新版本号 |

### 不兼容更新示例

当接口需要不兼容更新时：

```java
// 旧版本接口
@RestController
@RequestMapping("/v1/characters")  // 固定为v1
@Tag(name = "v1")                  // 固定标签
public class CharacterControllerV1 {
    // 旧版本实现
}

// 新版本接口
@RestController
@RequestMapping("/" + ApiConstant.LATEST + "/characters")
@Tag(name = ApiConstant.LATEST)
public class CharacterController {
    // 新版本实现
}
```

### 版本弃用

弃用旧版本时：
1. 标注 `@Deprecated` 注解
2. 在文档中说明弃用原因
3. 给出迁移建议
4. 设置弃用过渡期

## 数据库迁移规范

### Flyway命名规范

迁移脚本命名格式：

```
V1.x.x__description.sql
```

**组成部分：**

| 部分 | 说明 | 示例 |
|------|------|------|
| V | 版本标识（固定） | V |
| 1.x.x | 版本号 | 1.7.0 |
| __ | 双下划线分隔符 | __ |
| description | 描述（下划线分隔） | create_game_config |
| .sql | 文件扩展名 | .sql |

### 版本号规则

| 版本范围 | 用途 |
|----------|------|
| V1.0.x | 核心表创建 |
| V1.1.x | 管理员/扩展表 |
| V1.2.x | 功能扩展 |
| V1.3.x | 属性/配置表 |
| V1.4.x | 扭蛋/奖励表 |
| V1.5.x | 命令信息 |
| V1.6.x | 游戏数据 |
| V1.7.x | 配置/多语言 |
| V1.8.x | 配置更新 |
| V1.10.x | 自动封禁 |

### 脚本内容规范

#### 创建表脚本

```sql
-- V1.7.0__create_game_config.sql

-- 删除旧表（如果需要）
drop table if exists game_config;

-- 创建表
create table if not exists game_config
(
    id              bigint auto_increment primary key comment '自增id',
    config_type     varchar(32) not null comment '参数类型',
    config_sub_type varchar(32) comment '参数子类型',
    -- ...其他字段
) comment '游戏参数表';

-- 初始数据
insert into game_config (config_type, config_sub_type, config_code, config_value)
values ('world', '0', 'exp_rate', '1.0');
```

#### 更新脚本

```sql
-- V1.8.5__update_game_config_param.sql

-- 更新配置
UPDATE game_config 
SET config_value = '2.0' 
WHERE config_code = 'exp_rate';

-- 新增配置
INSERT INTO game_config (config_type, config_code, config_value)
VALUES ('world', 'new_param', '1.0');
```

### 迁移原则

1. **不可修改已执行脚本** - 已执行的脚本不能修改内容
2. **增量迁移** - 只添加新脚本，不修改旧脚本
3. **回滚机制** - 如需回滚，创建回滚脚本（如 V1.7.1__rollback_xxx.sql）
4. **原子性** - 每个脚本应该是原子操作

#### 包结构

```
org.gms/
├── ServerApplication.java    # 主入口
```

#### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | CharacterController |
| 方法名 | camelCase | getCharacterById |
| 常量 | UPPER_SNAKE_CASE | MAX_LEVEL |
| 变量 | camelCase | characterId |

```java

     */

        // 实现
    }
```

#### 使用Lombok

项目使用 Lombok 减少样板代码：


## Git提交规范

### 分支管理

| 分支 | 用途 |
|------|------|
| main | 主分支，稳定版本 |
| develop | 开发分支 |
| feature/* | 功能分支 |
| bugfix/* | Bug修复分支 |
| release/* | 发布分支 |

### 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 类型说明

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug修复 |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具相关 |

### 提交示例

```bash
# 新功能
feat(server): 添加转生系统

# Bug修复
fix(client): 修复中文输入卡门问题

# 文档
docs: 更新部署文档

# 重构
refactor(service): 重构角色服务层
```

### Pull Request规范

PR标题格式：

```
[类型] 简短描述
```

示例：
- `[feat] 添加转生系统`
- `[fix] 修复中文输入问题`

PR描述应包含：
1. 改动说明
2. 测试步骤
3. 影响范围

## 多语言规范

### 资源路径规范

BeiDou 支持多语言，资源路径分离：

| 资源类型 | 中文路径 | 英文路径 |
|----------|----------|----------|
| WZ文件 | wz-zh-CN/ | wz-en-US/ |
| 脚本 | script-zh-CN/ | script-en-US/ |

### 数据库多语言

`lang_resources` 表存储国际化：

```sql
-- 中文资源
INSERT INTO lang_resources (lang_type, lang_code, lang_value)
VALUES ('zh-CN', 'exp_rate', '经验倍率');

-- 英文资源
INSERT INTO lang_resources (lang_type, lang_code, lang_value)
VALUES ('en-US', 'exp_rate', 'exp rate');
```

### 服务端配置

```yaml
gms:
  service:
    language: zh-CN   # 或 en-US
```

### 前端国际化

使用 Vue I18n：

```typescript
// locale/zh-CN.ts
export default {
  character: '角色',
  level: '等级',
  // ...
};

// locale/en-US.ts
export default {
  character: 'Character',
  level: 'Level',
  // ...
};
```

## 安全规范

### 密码安全

1. 使用 bcrypt 加密存储
2. 开启 bcrypt_migration 配置
3. 生产环境禁止明文密码

```yaml
# application.yml
server:
  Safe:
    bcrypt_migration: true
```

### JWT安全

1. 生产环境必须修改 secret
2. 使用 HTTPS
3. 定期更换密钥

```yaml
jwt:
  secret: "your-custom-uuid"  # 必须修改
```

### API安全

1. 限流保护
2. Token验证
3. 权限控制

```yaml
gms:
  service:
    rate-limit:
      enabled: true
      limit: 100
      duration: 60000
```

### 数据库安全

1. 使用强密码
2. 限制远程访问
3. 定期备份

### 生产环境检查清单

| 项目 | 检查点 |
|------|--------|
| JWT密钥 | 已修改为自定义UUID |
| Swagger | 已关闭 |
| 限流 | 已开启 |
| HTTPS | 已配置 |
| 数据库密码 | 已使用强密码 |
| 默认账号密码 | 已修改 |

## 性能规范

### JVM配置

推荐生产环境JVM参数：

```bash
-Xms4g -Xmx8g -XX:+UseG1GC -XX:MaxGCPauseMillis=200
```

### 数据库优化

1. 使用连接池
2. 索引优化
3. 查询优化

```yaml
mybatis-flex:
  datasource:
    mysql:
      type: com.alibaba.druid.pool.DruidDataSource
```

### 缓存策略

对于频繁读取的数据：
1. 使用内存缓存
2. 定期刷新
3. 合理设置过期时间

### 网络优化

1. 使用 Undertow 替代 Tomcat
2. 合理设置超时时间
3. 连接池管理

## 测试规范

### 单元测试

使用 JUnit 5 + Mockito：

```java
@Test
void testGetCharacterById() {
    // Given
    int id = 1;
    
    // When
    Character character = service.getCharacterById(id);
    
    // Then
    assertNotNull(character);
    assertEquals(id, character.getId());
}
```

### 测试命名

测试方法命名：

```
test_<方法名>_should_<预期结果>_when_<条件>
```

示例：
- `test_getCharacterById_should_returnCharacter_when_idExists`
- `test_getCharacterById_should_throwException_when_idNotFound`
